==== Description

This output allows you to pull metrics from your logs and ship them to
opentsdb. Opentsdb is an open source tool for storing and graphing metrics.

[id="{version}-plugins-{type}s-{plugin}-host"]
===== `host` 

  * Value type is {logstash-ref}/configuration-file-structure.html#string[string]
  * Default value is `"localhost"`

The address of the opentsdb server.

[id="{version}-plugins-{type}s-{plugin}-metrics"]
===== `metrics` 

  * This is a required setting.
  * Value type is {logstash-ref}/configuration-file-structure.html#array[array]
  * There is no default value for this setting.

The metric(s) to use. This supports dynamic strings like %{source_host}
for metric names and also for values. This is an array field with key
of the metric name, value of the metric value, and multiple tag,values . Example:
[source,ruby]
    [
      "%{host}/uptime",
      %{uptime_1m} " ,
      "hostname" ,
      "%{host}
      "anotherhostname" ,
      "%{host}
    ]

The value will be coerced to a floating point value. Values which cannot be
coerced will zero (0)

[id="{version}-plugins-{type}s-{plugin}-port"]
===== `port` 

  * Value type is {logstash-ref}/configuration-file-structure.html#number[number]
  * Default value is `4242`

The port to connect on your graphite server.
